# 📊 C语言函数统计分析器

## 【这个项目是啥？】
一个专门用来分析C语言代码的工具，它能自动找出代码中的所有函数，并统计每个函数的长度，给出平均值、最大值、最小值、中位数等数据。

## 【它能做什么？】
- 📁 **扫描文件夹**: 自动扫描指定文件夹中的所有.c和.h文件
- 🔍 **识别函数**: 智能识别C语言函数定义，包括各种返回类型和参数形式
- 📊 **统计分析**: 计算函数长度的各种统计数据（平均值、中位数、最大值、最小值）
- 📈 **可视化展示**: 用漂亮的图表和表格展示分析结果
- 💡 **代码建议**: 根据函数长度给出重构建议，帮助改善代码质量
- ⚙️ **灵活配置**: 可以选择是否统计空行和注释行

## 【怎么使用它？】

### 1. 运行程序
在项目目录下运行：
```bash
cd /home/mutsumi/Workspace/tankGame/src/main/java
javac Game0_11/*.java
java Game0_11.Main
```

### 2. 设置参数
程序启动后会弹出一个设置对话框：
- 点击"浏览..."按钮选择要分析的C语言代码文件夹
- 选择编程语言（默认是C语言）
- 勾选是否要统计空行和注释行
- 点击"开始分析"按钮

### 3. 查看结果
分析完成后会显示一个结果窗口，包含4个标签页：
- **📈 统计概览**: 显示函数总数、平均长度等关键指标
- **📊 函数分布图**: 用直方图展示函数长度的分布情况
- **📋 函数详情**: 详细列出每个函数的信息（可排序）
- **💡 代码建议**: 给出具体的代码优化建议

## 【常见问题解答】

**Q: 为什么没有找到任何函数？**
A: 可能的原因：
- 选择的文件夹中没有.c或.h文件
- 函数定义格式不标准（比如函数定义跨多行）
- 文件编码问题

**Q: 什么样的函数长度是合理的？**
A: 一般建议：
- 理想情况：10-30行
- 可接受：30-50行
- 需要重构：超过50行
- 必须重构：超过100行

**Q: 空行和注释行应该算入函数长度吗？**
A: 这取决于你的需求：
- 如果想了解函数的视觉长度，应该勾选
- 如果只想统计纯代码量，可以取消勾选
